First: What exactly is TCP/IP?
    TCP/IP is a pair of protocols that tell computers how to send data across networks.
    It was designed by:
        Vint Cerf
        Bob Kahn
    And became official on ARPANET in 1983.

    TCP = Transmission Control Protocol
    Responsible for:
        Breaking data into packets
        Making sure nothing is lost
        Re-sending missing packets
        Putting packets back in correct order
    Think: ‚ÄúDid everything arrive safely?‚Äù

    IP = Internet Protocol
    Responsible for:
        Giving each computer an address (IP address)
        Deciding where packets should go
        Routing packets across routers
    Think: ‚ÄúWhere should this packet travel?‚Äù

In simple words:
      IP finds the destination.
      TCP guarantees safe delivery.
    Together = TCP/IP.
    This combo is why the Internet actually works reliably.

Now the important part:
    What was BEFORE TCP/IP?  
        Answer: NCP (Network Control Protocol)
        Early ARPANET (1970s) used NCP.

Problems with NCP
    NCP could:
         Communicate inside ARPANET
         But it could NOT:
             Connect different networks
             Scale globally
             Handle packet loss properly
             Support international expansion
          It assumed:
              ‚ÄúEveryone is on the same network.‚Äù
            That became a big limitation.

The Big Realization (late 1970s)
      Researchers wanted to connect:
          Military networks
          University networks
          Satellite networks
          Radio networks
      NCP completely failed here.
      So Cerf & Kahn created TCP/IP with a revolutionary idea:
          Any network ‚Üí any technology ‚Üí same protocol.
      This is called:
          Internetworking
      (That‚Äôs literally where the word Internet comes from.)

1983 ‚Äî The Historic Switch
      ARPANET officially dropped NCP and adopted TCP/IP.
      This moment is considered:
          The birth of the modern Internet
      After that:
          Universities joined
          Countries joined
          Commercial networks joined
      Boom ‚Üí global Internet.

What is the difference between TCP and UDP?

    TCP (Transmission Control Protocol) ‚Üí Reliable, ordered, slow but safe
    UDP (User Datagram Protocol) ‚Üí Fast, lightweight, but no guarantees
    TCP = Registered post (with tracking)
    UDP = Normal post (just throw and go)

TCP (Transmission Control Protocol)
      What TCP does:
          Establishes connection first (3-way handshake)
          Sends data in order
          Detects lost packets
          Retransmits missing data
          Guarantees delivery
      Result:
          Slower
          More overhead
          Extremely reliable
      Used when correctness matters:
          Websites (HTTP/HTTPS)
          Email
          File downloads
          APIs
          Banking systems
      Example:
          If you download a PDF ‚Äî even 1 missing byte is unacceptable ‚Üí TCP.

UDP (User Datagram Protocol)
    What UDP does:
      No connection setup
      No delivery guarantee
      No reordering
      No retransmission
    Just:
      Send ‚Üí forget.
    Result:
      Very fast
      Very low overhead
      Can lose data
    Used when speed matters more than perfection:
      Video streaming
      Online games
      Voice calls
      Live broadcasts
    Example:
      In a video call, losing 1 frame is okay ‚Äî waiting for it is not ‚Üí UDP.

| Feature        | TCP              | UDP                 |
| -------------- | ---------------- | ------------------- |
| Connection     | Yes              | No                  |
| Reliability    | Guaranteed       | Not guaranteed      |
| Order          | Maintained       | Not maintained      |
| Speed          | Slower           | Faster              |
| Retransmission | Yes              | No                  |
| Overhead       | High             | Low                 |
| Typical use    | Web, APIs, Files | Games, Video, Voice |

What is overhead?
    Networking overhead is the additional information, processing, or time required by protocols and systems to successfully deliver data over a network, beyond the actual payload.

Real-world analogy (easy to remember)
  TCP:
    Teacher sending exam papers and waiting for acknowledgment from every student.
  UDP:
    Announcer speaking on loudspeaker ‚Äî whoever hears it, hears it.

TCP prioritizes reliability; UDP prioritizes speed.

Important practical mapping:
    REST APIs ‚Üí TCP
    Database connections ‚Üí TCP
    Video streaming ‚Üí UDP
    Multiplayer games ‚Üí UDP
